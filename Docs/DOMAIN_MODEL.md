# ドメインモデル仕様書

## 概要

達成感を味わいながら心の健康をサポートするアプリのドメインモデルを定義します。
このドメインモデルは、DDD（ドメイン駆動設計）の原則に基づいて設計されています。

当初はTODO機能のみを想定していましたが、現在は認知行動療法（CBT）やコーピングリストなど、心理療法機能も統合しています。

## プロダクト目的

- 日々の達成感を味わえるTODO管理機能の提供
- 過去の完了TODOを振り返ることで達成感を得られる
- ゴール設定ではなく、作業の積み重ねを実感できる
- ストレスや不安への対処法を管理できるコーピングリスト
- 認知の歪みに気づき、適応的な思考を身につけるコラム法（CBT）

## ユビキタス言語

プロジェクトに関わる全ての人が使用する共通の用語を定義します。

### TODO管理機能の用語

| 用語 | 定義 |
|------|------|
| **TODO** | やるべきこと |
| **完了** | TODOが終わった状態 |
| **タグ** | TODOを分類するラベル（例：個人開発、家事、育児） |
| **難易度** | TODOの大きさ（大・中・小の3段階） |
| **達成感** | 完了したTODOを見て感じる満足感 |
| **完了TODO** | 完了したTODOのこと |
| **クイックタスク** | 頻繁に登録するタスクのテンプレート（家事・育児など） |

### コーピング機能の用語

| 用語 | 定義 |
|------|------|
| **コーピング（Coping）** | ストレスや不安に対処するための方法やスキル |
| **コーピングリスト** | 利用可能な対処法の一覧 |
| **コーピングタグ** | コーピング方法を分類するラベル（例：身体的、認知的、社会的） |
| **お気に入りポイント** | コーピング方法の効果度を示す指標（高いほど効果的） |

### コラム法（CBT）機能の用語

| 用語 | 定義 |
|------|------|
| **コラム法（7カラム法）** | 認知行動療法の技法の一つ |
| **状況（Situation）** | 問題となった出来事や場面 |
| **感情（Mood）** | その状況で感じた感情 |
| **自動思考（Automatic Thought）** | 自動的に浮かんだ考えやイメージ |
| **根拠（Evidence）** | 自動思考を支持する事実 |
| **反証（Counter Evidence）** | 自動思考に反する事実 |
| **適応的思考（Adaptive Thought）** | バランスの取れた合理的な考え方 |
| **現在の感情（Current Mood）** | 認知再構成後の感情 |
| **認知の歪み** | 非合理的で偏った思考パターン |
| **認知再構成** | 否定的な思考をより適応的な思考に置き換えるプロセス |

## ドメインモデル

### エンティティ

#### TODO
**説明**: アプリケーションの中核となるエンティティ

**属性**:
- ID（一意識別子）
- タイトル
- 説明（オプション）
- 作成日時
- 更新日時
- 完了日時（完了時のみ）
- 完了状態（未完了・完了）
- タグ（複数）
- 難易度

**振る舞い**:
- 作成
- 編集
- 削除
- 完了
- 一覧表示

#### タグ
**説明**: TODOを分類するためのラベル

**属性**:
- ID（一意識別子）
- 名前
- 作成日時

**振る舞い**:
- 作成
- 編集
- 削除

#### 難易度
**説明**: TODOの大きさを表すエンティティ

**属性**:
- ID（一意識別子）
- 名前（大・中・小）
- ポイント（3・2・1）
- 色（将来拡張）
- 説明（将来拡張）
- 作成日時

**振る舞い**:
- 作成
- 編集
- 削除

#### コーピング
**説明**: ストレスや不安に対処するための方法を表すエンティティ

**属性**:
- ID（一意識別子）
- 内容（対処法の説明）
- お気に入りポイント（効果度、デフォルト0）
- 作成日時
- 更新日時
- コーピングタグ（複数）

**振る舞い**:
- 作成
- 編集（内容とタグの更新）
- 削除
- ポイント増加（効果があった時）
- ポイント減少

#### コーピングタグ
**説明**: コーピング方法を分類するためのラベル

**属性**:
- ID（一意識別子）
- 名前
- 作成日時

**振る舞い**:
- 作成
- 編集
- 削除

#### コラム（Column）
**説明**: 認知行動療法の7カラム法の記録を表すエンティティ

**属性**:
- ID（一意識別子）
- 状況（Situation）
- 感情（Mood）
- 自動思考（Automatic Thought）
- 根拠（Evidence）
- 反証（Counter Evidence）
- 適応的思考（Adaptive Thought）
- 現在の感情（Current Mood）
- 作成日時
- 更新日時

**振る舞い**:
- 作成
- 閲覧
- 削除（編集は現時点では実装していない）

#### クイックタスク（QuickTask）
**説明**: 頻繁に登録するタスクのテンプレートを表すエンティティ

**属性**:
- ID（一意識別子）
- 内容（タスクのテンプレート文）
- 難易度（オプション、デフォルト設定用）
- タグ（複数、オプション、デフォルト設定用）
- 作成日時
- 更新日時

**振る舞い**:
- 作成
- 編集
- 削除
- TODOへの転記（TODOフォームに内容・難易度・タグをコピー）

### 値オブジェクト

#### 完了状態
**説明**: TODOの完了状況を表す値

**値**:
- 未完了
- 完了

#### TODOコンテンツ
**説明**: TODOの内容を表す値オブジェクト

**検証ルール**:
- 空文字列不可
- 最大文字数制限

#### コーピングコンテンツ
**説明**: コーピング方法の内容を表す値オブジェクト

**検証ルール**:
- 空文字列不可
- 最大文字数制限

#### クイックタスクコンテンツ
**説明**: クイックタスクの内容を表す値オブジェクト

**検証ルール**:
- 空文字列不可
- 最大200文字

### エンティティ間の関係

#### TODO と タグ
- **関係**: 多対多
- **説明**: 1つのTODOに複数のタグを付けることができ、1つのタグを複数のTODOで使用できる

#### TODO と 難易度
- **関係**: 多対1
- **説明**: 1つのTODOに1つの難易度が設定される。複数のTODOが同じ難易度を参照できる

#### TODO と 完了状態
- **関係**: 1対1
- **説明**: 1つのTODOに1つの完了状態が設定される

#### コーピング と コーピングタグ
- **関係**: 多対多
- **説明**: 1つのコーピングに複数のタグを付けることができ、1つのタグを複数のコーピングで使用できる

#### クイックタスク と タグ
- **関係**: 多対多
- **説明**: 1つのクイックタスクに複数のタグを事前設定できる。TODOへの転記時に使用

#### クイックタスク と 難易度
- **関係**: 多対1（オプション）
- **説明**: 1つのクイックタスクに1つの難易度を事前設定できる。TODOへの転記時に使用

### ドメインサービス

#### 完了TODO集計サービス
**説明**: 完了したTODOの集計を行う

**機能**:
- 期間別完了TODO数の集計
- タグ別完了TODO数の集計
- 難易度別完了TODO数の集計

#### 達成感算出サービス
**説明**: 完了TODOから達成感スコアを算出する

**機能**:
- 完了TODOの難易度ポイント合計
- 期間別達成感スコア

#### TODO検索サービス
**説明**: 複数条件でのTODO検索・絞り込みを行う

**機能**:
- 複数条件での検索・絞り込み
- 年月日別での完了TODO絞り込み
- タグ別での完了TODO絞り込み

## 満たすべき要件

### 実装すべき機能

#### TODO管理機能
1. **基本的なTODO機能**
   - 作成、編集、削除、完了、一覧表示

2. **タグ作成機能**
   - TODOに追加できる分類情報

3. **難易度追加機能**
   - 大中小の3段階でタスクにレベル付け

4. **完了TODO閲覧機能**
   - 年月日別やタグ別で絞り込み
   - 総完了件数および難易度ごとの完了件数を表示

5. **クイックタスク機能**
   - 頻繁に登録するタスクのテンプレート管理
   - 作成、編集、削除、一覧表示
   - TODOページでワンタップで入力フォームに転記

#### コーピングリスト機能
1. **コーピング方法の管理**
   - 作成、編集、削除、一覧表示

2. **お気に入りポイント機能**
   - ポイントの増減
   - ポイント順でのソート表示

3. **コーピングタグ機能**
   - タグの作成、一覧表示
   - タグによるコーピングの分類

#### コラム法（CBT）機能
1. **7カラム記録の管理**
   - 作成、閲覧、削除

2. **7つの項目の記録**
   - 状況、感情、自動思考、根拠、反証、適応的思考、現在の感情

3. **記録の一覧表示**
   - 過去の認知再構成記録の振り返り

### 実装しない機能

#### TODO管理機能において
1. **達成したいゴールの設定**
   - 目的は達成感を味わうことであり、達成できなかった時のネガティブ感情を避けるため

2. **平均タスク消化数の表示**
   - 比較して少ないとモチベーションが下がるため

#### 心理療法機能において
1. **専門的な診断や治療提案**
   - あくまでセルフケアツールとしての位置づけ

2. **他者との比較機能**
   - ネガティブな感情を引き起こす可能性があるため

3. **強制的な通知やリマインダー**
   - ユーザーのペースを尊重するため
