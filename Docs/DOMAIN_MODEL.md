# ドメインモデル仕様書

## 概要

達成感を味わえるTODOリストアプリのドメインモデルを定義します。
このドメインモデルは、DDD（ドメイン駆動設計）の原則に基づいて設計されています。

## プロダクト目的

- 日々の達成感を味わえるTODOリストアプリの開発
- 過去の完了TODOを振り返ることで達成感を得られる
- ゴール設定ではなく、作業の積み重ねを実感できる

## ユビキタス言語

プロジェクトに関わる全ての人が使用する共通の用語を定義します。

| 用語 | 定義 |
|------|------|
| **TODO** | やるべきこと |
| **完了** | TODOが終わった状態 |
| **タグ** | TODOを分類するラベル（例：個人開発、家事、育児） |
| **難易度** | TODOの大きさ（大・中・小の3段階） |
| **達成感** | 完了したTODOを見て感じる満足感 |
| **完了TODO** | 完了したTODOのこと |

## ドメインモデル

### エンティティ

#### TODO
**説明**: アプリケーションの中核となるエンティティ

**属性**:
- ID（一意識別子）
- タイトル
- 説明（オプション）
- 作成日時
- 更新日時
- 完了日時（完了時のみ）
- 完了状態（未完了・完了）
- タグ（複数）
- 難易度

**振る舞い**:
- 作成
- 編集
- 削除
- 完了
- 一覧表示

#### タグ
**説明**: TODOを分類するためのラベル

**属性**:
- ID（一意識別子）
- 名前
- 作成日時

**振る舞い**:
- 作成
- 編集
- 削除

#### 難易度
**説明**: TODOの大きさを表すエンティティ

**属性**:
- ID（一意識別子）
- 名前（大・中・小）
- ポイント（3・2・1）
- 色（将来拡張）
- 説明（将来拡張）
- 作成日時

**振る舞い**:
- 作成
- 編集
- 削除

### 値オブジェクト

#### 完了状態
**説明**: TODOの完了状況を表す値

**値**:
- 未完了
- 完了

### エンティティ間の関係

#### TODO と タグ
- **関係**: 多対多
- **説明**: 1つのTODOに複数のタグを付けることができ、1つのタグを複数のTODOで使用できる

#### TODO と 難易度
- **関係**: 多対1
- **説明**: 1つのTODOに1つの難易度が設定される。複数のTODOが同じ難易度を参照できる

#### TODO と 完了状態
- **関係**: 1対1
- **説明**: 1つのTODOに1つの完了状態が設定される

### ドメインサービス

#### 完了TODO集計サービス
**説明**: 完了したTODOの集計を行う

**機能**:
- 期間別完了TODO数の集計
- タグ別完了TODO数の集計
- 難易度別完了TODO数の集計

#### 達成感算出サービス
**説明**: 完了TODOから達成感スコアを算出する

**機能**:
- 完了TODOの難易度ポイント合計
- 期間別達成感スコア

#### TODO検索サービス
**説明**: 複数条件でのTODO検索・絞り込みを行う

**機能**:
- 複数条件での検索・絞り込み
- 年月日別での完了TODO絞り込み
- タグ別での完了TODO絞り込み

## 満たすべき要件

### 実装すべき機能
1. **基本的なTODO機能**
   - 作成、編集、削除、完了、一覧表示

2. **タグ作成機能**
   - TODOに追加できる分類情報

3. **難易度追加機能**
   - 大中小の3段階でタスクにレベル付け

4. **完了TODO閲覧機能**
   - 年月日別やタグ別で絞り込み
   - 総完了件数および難易度ごとの完了件数を表示

### 実装しない機能
1. **達成したいゴールの設定**
   - 目的は達成感を味わうことであり、達成できなかった時のネガティブ感情を避けるため

2. **平均タスク消化数の表示**
   - 比較して少ないとモチベーションが下がるため
