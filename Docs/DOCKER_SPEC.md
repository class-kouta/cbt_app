# Docker 環境仕様書

## 1. 概要

本プロジェクトのローカル開発環境をDockerで構築するための仕様書です。
`docker-compose` を使用して、アプリケーション実行に必要なコンテナ群を管理します。

## 2. ディレクトリ構成

`docker-compose.yml` はプロジェクトルートに配置します。
各サービス（Nginx, PHP）の設定ファイルは、`docker` ディレクトリ内にサービスごとのサブディレクトリ（`nginx/`, `php/`など）を作成して管理します。
この構成には以下のメリットがあります。

-   **可読性と管理性の向上:** サービスごとに関連ファイルがまとまるため、どこに何の設定があるか直感的に把握しやすくなります。
-   **拡張性の確保:** 将来的にRedisや他のコンテナを追加する際も、新しいサービス用のディレクトリを追加するだけで済み、プロジェクトがスケールしても構成の複雑化を防ぎます。

これにより、長期的なプロジェクトのメンテナンス性が向上します。


```
.
├── docker-compose.yml
├── docker
│   ├── nginx
│   │   └── default.conf
│   └── php
│       └── Dockerfile
├── ... (既存のLaravelプロジェクトファイル)
```

## 3. コンテナ構成

`docker-compose.yml` で以下の3つのサービスを定義します。

1.  **`app` (PHP-FPM):** Laravelアプリケーションの本体（PHPコード）を動かすための専門コンテナ
2.  **`web` (Nginx):** 外部からのアクセスを受け付け、リクエストをPHPコンテナに振り分ける受付係
3.  **`db` (PostgreSQL):** アプリケーションのデータを安全に保管するデジタル金庫

---

### 3.1. app (PHP-FPM) サービス

Laravelアプリケーションの本体（PHPコード）を動かすための専門コンテナです。

-   **コンテナ名:** `todo-app`
    -   **解説:** Docker環境内でこのPHPコンテナを識別するための名前です。`docker exec` などでコンテナを直接操作する際に利用します。

-   **ビルドコンテキスト:** `build: { context: ./docker/php, dockerfile: Dockerfile }`
    -   **解説:** `./docker/php` ディレクトリにある `Dockerfile` を設計図として、このコンテナを構築（ビルド）します。

-   **ボリュームマウント:** `volumes: ['./:/var/www']`
    -   **解説:** ローカルPCのプロジェクトルート (`./`) をコンテナ内の `/var/www` に同期（マウント）します。これにより、ローカルでコードを修正すると即座にコンテナ内に反映され、開発効率が向上します。

-   **依存関係:** `depends_on: ['db']`
    -   **解説:** `db`コンテナが起動してから`app`コンテナが起動するように順序を制御します。これにより、アプリケーション起動時のデータベース接続エラーを防ぎます。

#### `docker/php/Dockerfile` の仕様

`app` コンテナの具体的な組立指示書です。

-   **ベースイメージ:** `php:8.4-fpm`
    -   **解説:** Debian Linux上で動く、バージョン8.4のPHP-FPM環境を土台とします。Alpine版と比較してイメージサイズは大きくなりますが、事前コンパイル済みのパッケージを利用できるため、ビルド時間を大幅に短縮できます。

-   **インストールするPHP拡張機能:**
    -   **解説:** TODOリストアプリに必要なPHPの追加機能（プラグイン）をインストールします。
    -   **主な拡張機能:** `pdo_pgsql`, `pgsql` (PostgreSQL接続用), `bcmath` (高精度計算), `zip` (アーカイブ処理)
    -   **除外した拡張機能:** `gd` (画像処理) - TODOリストアプリでは画像処理が不要なため、ビルド時間短縮のため除外

-   **Composerのインストール:**
    -   **解説:** PHPのパッケージ管理ツールである`Composer`を公式インストーラーを使用してインストールします。Laravel本体やその他ライブラリの依存関係を解決するために必須です。
    -   **インストール方法:** `curl`を使用して公式インストーラーをダウンロード・実行する効率的な方法を採用

-   **作業ディレクトリ:** `WORKDIR /var/www`
    -   **解説:** コンテナ内でのコマンド実行時のデフォルトの場所を `/var/www` に設定します。ボリュームマウント先と一致させることで、`composer install` などのコマンド実行を簡略化します。

### 3.2. web (Nginx) サービス

外部からのアクセスを受け付ける窓口となり、リクエストをPHPコンテナに振り分ける受付係の役割を担います。

-   **コンテナ名:** `todo-web`
    -   **解説:** Docker環境内でこのNginxコンテナを識別するための名前です。

-   **イメージ:** `image: nginx:1.25-alpine`
    -   **解説:** Docker Hubで公開されている公式の `Nginx` イメージ（バージョン1.25, alpineベース）を利用します。プロジェクト固有のビルドが不要なため、完成品のイメージをそのまま使います。

-   **ポートフォワーディング:** `ports: ['8080:80']`
    -   **解説:** ローカルPCの`8080`番ポートへのアクセスを、コンテナの`80`番ポート（HTTP標準ポート）に転送します。これにより、ブラウザから `http://localhost:8080` でアクセス可能になります。

-   **ボリュームマウント:**
    -   `./:/var/www`: `app`コンテナと共有し、静的ファイルへのアクセスを可能にします。
    -   `./docker/nginx/default.conf:/etc/nginx/conf.d/default.conf`: Nginxのカスタム設定ファイルをコンテナ内に配置し、デフォルト設定を上書きします。`/etc/`ディレクトリはNginxコンテナ内にプリインストールされています。

-   **依存関係:** `depends_on: ['app']`
    -   **解説:** `app`コンテナが起動してから`web`コンテナが起動するように順序を制御します。リクエストの転送先が必ず存在するようにし、`502 Bad Gateway` エラーを防ぎます。

#### `docker/nginx/default.conf` の仕様

`web` コンテナの具体的な行動マニュアルです。

-   **リッスンポート:** `listen 80;`
    -   **解説:** コンテナ内の`80`番ポートでリクエストを待ち受けます。

-   **ドキュメントルート:** `root /var/www/public;`
    -   **解説:** Webサイトの公開基点を、Laravelの標準構成に合わせて`/var/www/public`に設定します。

-   **PHPへのリクエスト転送:** `fastcgi_pass app:9000;`
    -   **解説:** `.php`で終わるリクエストを受け取ると、その処理を`app`サービス（PHPコンテナ）が待ち受ける`9000`番ポート（PHP-FPMの標準ポート）に転送します。

### 3.3. db (PostgreSQL) サービス

アプリケーションのデータを安全かつ永続的に保管するデジタル金庫です。

-   **コンテナ名:** `todo-db`
    -   **解説:** `app`コンテナが接続する際のホスト名として使われる、データベースコンテナの名前です。

-   **イメージ:** `image: postgres:15-alpine`
    -   **解説:** Docker Hubで公開されている公式の `PostgreSQL` イメージ（バージョン15, alpineベース）を利用します。

-   **ポートフォワーディング:** `ports: ['5432:5432']`
    -   **解説:** ローカルPCのデータベース管理ツールからコンテナ内のPostgreSQL（標準ポート`5432`）に接続するための「のぞき窓」です。

-   **環境変数:** `environment: { POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD }`
    -   **解説:** コンテナの初回起動時に、データベース名・ユーザー名・パスワードを初期設定するために利用します。これらの値はホストの`.env`ファイルから読み込まれます。

-   **ボリューム:** `volumes: ['todo_and_done_list_db_data:/var/lib/postgresql/data']`
    -   **解説:** データベースの物理ファイルを、Dockerが管理する「名前付きボリューム (`todo_and_done_list_db_data`)」に保存します。これにより、コンテナを削除してもデータが消えずに永続化されます。

## 4. ネットワーク

-   `docker-compose` によってデフォルトで作成されるブリッジネットワークを使用します。各サービスはサービス名をホスト名として相互に通信可能です。

## 5. .env ファイル

Laravelの `.env` ファイルでデータベース接続情報を以下のように設定します。

```env
DB_CONNECTION=pgsql
DB_HOST=db
DB_PORT=5432
DB_DATABASE=todo_and_done_list
DB_USERNAME=sail
DB_PASSWORD=password
```

これらの値は、`docker-compose.yml`の`db`サービスの環境変数と一致させる必要があります。

## 6. ボリューム定義

`docker-compose.yml`のトップレベルで名前付きボリュームを定義します。

```yaml
volumes:
  todo_and_done_list_db_data:
```

## 7. 環境構築手順

Docker環境を構築する際の基本的な手順：

1. **コンテナのビルドと起動**
   ```bash
   docker compose up -d --build
   ```

2. **依存関係のインストール**
   ```bash
   docker compose exec app composer install
   ```

3. **アプリケーションキーの生成**
   ```bash
   docker compose exec app php artisan key:generate
   ```

4. **データベースマイグレーション**
   ```bash
   docker compose exec app php artisan migrate
   ```

5. **設定キャッシュの生成**
   ```bash
   docker compose exec app php artisan config:cache
   ```

これらの手順により、`http://localhost:8080`でLaravelアプリケーションにアクセス可能になります。
