# 開発プロダクト

**誰でも手軽に使えるメンタルセルフケアアプリ**

## プロダクト発案理由

- 有名なコラム法以外にも、いろんなセルフケアを試せるアプリが欲しかった
- 有名なアプリは高価格で利用のハードルが高いため、リーズナブルなアプリを作りたかった
- 初心者でもうまく思考を整理できる工夫（例：コラム法の書き方がわからない場合に、AIが例文を出してくれる等）や、メンタルが落ち込んでやる気がないときでもアプリを利用できる工夫（例：コラム法の必須項目を「状況」だけにして、残りの項目は後で記載できるようにするなど）があるアプリを作りたかった。

## プロダクト概要

- コラム法
- コーピングリスト
- 筆記開示
- 問題解決法
- 他にも今後多様な機能を実装予定

### 1. コーピングリスト機能

#### 概要
ストレスや不安に対処するための方法（コーピングスキル）を管理する機能

#### 主な機能
- コーピング方法の作成、編集、削除、一覧表示
- お気に入りポイント機能（効果的な方法を上位表示）
- タグによる分類と検索
- 例：深呼吸、散歩、音楽を聴く、友人に相談するなど

### 2. コラム法機能（7カラム法）

#### 概要
認知行動療法（CBT）の代表的な技法である7カラム法を実装

#### 7つの項目
1. **状況（Situation）**: 何が起こったか
2. **感情（Mood）**: どんな気持ちになったか
3. **自動思考（Automatic Thought）**: どんな考えが浮かんだか
4. **根拠（Evidence）**: その考えを支持する事実
5. **反証（Counter Evidence）**: その考えに反する事実
6. **適応的思考（Adaptive Thought）**: バランスの取れた考え方
7. **現在の感情（Current Mood）**: 考え直した後の気持ち

#### 目的
- 思考パターンの可視化
- 認知の歪みへの気づき
- より適応的な思考への転換

## 本プロダクトでやらないこと

- 高すぎる利用料
- 独自で考えたセルフケア方法の実装（本アプリでは一般に認めらている心理療法のみ取り扱うこととしているため）

# 開発の進め方

## 概要

- 知見のあるLaravelで開発する
- フロントエンドは最初はbladeおよびバニラjsで開発し、後からReactなどのモダンJSフレームワークに置き換える
- 最終的にReact Nativeを使った web view アプリの開発をゴールとする

## 開発要件

### バックエンドとフロントエンドは二つに分ける

- 明確な役割分担：LaravelはAPI、ReactはUIと分離
- React Native連携容易：Webとモバイルで同じAPIを再利用できる。
- 高い柔軟性：フロントとバックエンドを個別にスケール・技術変更できる。

#### バックエンド：ビルトインサーバーではなくDockerを使う理由

- 環境の一貫性: Dockerを使うと、開発環境（PHPのバージョン、拡張機能、Webサーバー、データベース）を本番環境（Herokuなど）とほぼ同じ構成にできます。これにより、「ローカルでは動くのにデプロイしたら動かない」といった問題を減らせます。ビルトインサーバーでは、このような環境の一貫性を保つのが難しいです。

- 環境構築の簡素化と管理: PHP、Webサーバー（Nginxなど）、データベース（PostgreSQL）といった複数のソフトウェアを、個別にローカルPCにインストールして設定するのは面倒で、他のプロジェクトとの競合も起こりえます。Dockerを使えば、これらの環境をコンテナとして隔離し、docker-compose upコマンド一つでまとめて起動・停止できます。

- CI/CDへの移行がスムーズ: DockerはCI/CDパイプラインの基盤となることが多く、ローカルでDockerを使う経験は、将来の自動テストや自動デプロイの設定に直結します。

### `docker-compose.yml`はLaravelルートに、その他Docker関連ファイルは置く

- **コマンド実行の容易さ:** `docker-compose.yml`をプロジェクトルートに置くことで、`docker-compose up`コマンドをすぐに実行できます。
- **設定のシンプルさ:** `docker-compose.yml`からのボリュームマウント等のパス指定が簡単になります。
- **管理のしやすさ:** NginxやPHPのDockerfileといった個別設定は`docker/`ディレクトリにまとめることで、ルートディレクトリをクリーンに保ちます。
- **開発環境の再現性:** `git clone`してすぐに開発環境を立ち上げられるため、新しい開発者の参加がスムーズになります。
